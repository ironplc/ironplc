# Plan: Update fast-json-patch to 3.1.1 by replacing ajv-cli

## Context

`fast-json-patch@2.2.1` is currently installed in the VSCode extension's dependency tree and needs to be upgraded to `3.1.1` to address a security risk. It enters the tree purely as a transitive dependency of `ajv-cli@5.0.0` — `fast-json-patch` is NOT used anywhere in the IronPLC source itself.

The root cause is that `ajv-cli@5.0.0` (the latest release, published Sept 2022) pins `fast-json-patch: ^2.0.0` and has not been updated in roughly three years, so there is no maintained upstream fix. Rather than mask the problem with an `npm overrides` shim that keeps an abandoned package in the tree, the cleanest fix is to **remove `ajv-cli` entirely** and replace its single use with a small Node script that calls the `ajv` library directly. This:

- Eliminates `fast-json-patch` from the dependency tree completely (no override needed — `ajv` itself does not depend on it).
- Drops an unmaintained dependency (security + supply-chain win).
- Keeps the toolchain pure-Node — no new languages or external tools.

## Scope of the existing ajv-cli usage

`ajv-cli` is invoked in exactly one place — the `validate-grammars` npm script in `integrations/vscode/package.json:255`:

```
"validate-grammars": "ajv validate -s schemas/tmlanguage.json -d \"syntaxes/*.tmLanguage.json\""
```

It validates two TextMate grammar files — `syntaxes/61131-3-st.tmLanguage.json` and `syntaxes/plcopen-xml.tmLanguage.json` — against `schemas/tmlanguage.json` (a standard JSON Schema draft-04 file). The script is wired into `lint` (`package.json:256`), which runs in `.github/workflows/partial_vscode_extension.yaml`.

No other invocation of `ajv-cli` exists in the repo.

## Approach

### 1. Add a small validation script

Create `integrations/vscode/scripts/validate-grammars.js` — a plain Node.js CommonJS script following the pattern already established by `scripts/generate-problems.js` (`scripts/generate-problems.js:1-16`). The script loads the schema, compiles it with Ajv (draft-04 mode), iterates over the grammar files, and exits non-zero with a formatted error list on failure.

Key points:
- Use `ajv` (major version `^8`) with `ajv-draft-04` for the draft-04 schema (the schema declares `"$schema": "http://json-schema.org/schema#"`, which is draft-04 in ajv-cli's default behavior).
- Use `glob` (already a direct devDependency at `^11` per `package.json:278`) to expand `syntaxes/*.tmLanguage.json`.
- Print Ajv error output via `ajv.errorsText()` or by iterating `validate.errors`, keyed by file path. Exit with code 1 on any validation failure.
- ~40 lines total.

### 2. Swap the dependency

In `integrations/vscode/package.json`:

- Remove `"ajv-cli": "^5"` from `devDependencies` (`package.json:274`).
- Add `"ajv": "^8.17.1"` and `"ajv-draft-04": "^1.0.0"` to `devDependencies` (alphabetical order). These are direct, actively maintained packages.
- Update the `validate-grammars` script at `package.json:255` to:
  ```
  "validate-grammars": "node scripts/validate-grammars.js"
  ```

### 3. Regenerate the lock file

Run `npm install` inside `integrations/vscode/` to update `package-lock.json`. Expected diff:

- Remove `node_modules/ajv-cli` and its entire sub-tree.
- Remove `node_modules/fast-json-patch` (`package-lock.json:3811`) and the nested `fast-json-patch/node_modules/fast-deep-equal@2.0.1` (`package-lock.json:3824`).
- Remove other ajv-cli-specific transitives that aren't pulled in by anything else (e.g. `json-schema-migrate`, `minimist@1.x` if not required elsewhere).
- Add a top-level `node_modules/ajv-draft-04`. `ajv` itself is almost certainly already present transitively (through `eslint` and others); confirm via `npm ls ajv` after install.
